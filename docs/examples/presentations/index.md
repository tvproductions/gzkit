# Video Tutorial Series: Governed Software Development

## CIDM 6330/6395 — Software Engineering Governance

---

## Series Overview

This six-part video series teaches you how to govern your software projects
using a structured, template-driven approach. You will learn to write Product
Requirements Documents, Architecture Decision Records, and Task Briefs that
keep you in the architect's seat — even when AI writes the code.

**Running example:** A reading list tracker CLI built from scratch through
the full governance cycle.

---

## Parts

| Part | Title | Duration | What You'll Learn |
|------|-------|----------|-------------------|
| 1 | [Why Governed Development?](part1-script.md) | ~10 min | The Degradation Pattern, Zero Doctrine, and why governance matters |
| 2 | [Product Requirements Documents](part2-script.md) | ~12 min | How to define what you're building and why, using the PRD template |
| 3 | [Architecture Decision Records](part3-script.md) | ~12 min | How to record design decisions before implementation |
| 4 | [Task Decomposition](part4-script.md) | ~12 min | Breaking features into small, testable tasks with the task template |
| 5 | [Implementation and Verification](part5-script.md) | ~12 min | Writing code, testing, and verifying acceptance criteria |
| 6 | [Putting It Together](part6-script.md) | ~10 min | The full cycle in review, file structure, and gzkit CLI preview |

**Total series:** ~68 minutes

---

## How to Use This Series

1. **Watch each part in order** — concepts build on each other
2. **Pause and do the activities** — each part ends with a hands-on prompt
3. **Use the templates** — copy from `docs/examples/templates/` into your project
4. **Reference the guides** — the written guides go deeper than the videos

---

## Prerequisites

- Familiarity with Python (basic syntax, functions, classes)
- A text editor (VS Code recommended)
- Git installed and configured
- Python 3.13+ with `uv` (for Part 5+)
